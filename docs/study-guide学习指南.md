**数据流向说明：**
1. 用户通过前端操作 → 发送 HTTP 请求到 Spring Boot
2. Spring Boot Controller 接收请求 → 调用 Service 层
3. Service 层根据需求：查数据库(MyBatis-Plus) 或 调用 AI 服务(RestTemplate)
4. AI 服务接到请求后 → 使用对应算法模型处理 → 返回 JSON 结果
5. 结果最终返回给前端展示

## 3. 各模块详细介绍

### 3.1 原有功能模块（不修改）

- **班级管理**：CRUD 操作，学员按班级分组
- **学员管理**：学员信息增删改查、违纪处理
- **数据统计**：学员学历分布、班级人数统计

### 3.2 新增 AI 功能模块

#### 成绩管理（Performance）

- 添加学员各科成绩
- 查询历史成绩
- **核心亮点**：调用 AI 预测期末成绩

#### 学员反馈（Feedback）

- 学员提交学习反馈
- 情感分析（AI 评分反馈的积极性/消极性）
- 班级情感趋势监控

#### AI 分析报告（Report 增强）

- 成绩预测报告：基于线性回归的趋势分析
- 聚类分析报告：KMeans 将学员分为不同学习群体
- 班级洞察报告：综合多维度数据分析

#### 个性化推荐（Recommend）

- 课程推荐：根据成绩弱点推荐学习内容
- 方法推荐：推荐高效学习方法
- 资料推荐：推荐书籍/视频/刷题平台

### 3.3 AI 模型详解

#### 成绩预测模型（线性回归）

```
特征： [课程序号, 历史平均分, 最近一次成绩]
      → 预测：下一门课程分数
```

- **输入**：学员按时间排序的各科成绩
- **算法**：LinearRegression（scikit-learn）
- **输出**：预测分数 + 置信度(R²) + 趋势分析
- **边界情况**：数据不足时用简单平均代替

#### 聚类模型（KMeans）

```
特征： [平均分, 成绩波动, 违纪次数, 出勤率]
      → 聚类：优秀组 / 中等组 / 提升组
```

- **输入**：全班学员的特征数据
- **算法**：KMeans (k=3)
- **输出**：分组结果 + 各组成员 + 群体特征描述
- **边界情况**：学员不足时用简单规则分组

#### 情感分析模型（基于词典）

```
文本 → jieba分词 + 情感词典匹配
      → 计算得分：-1.0(消极) ~ 1.0(积极)
```

- **核心**：积极词库(50+) + 消极词库(30+)
- **加权**：程度副词(非常→1.5x)、否定词反转
- **输出**：情感得分、极性、匹配到的关键词

#### 推荐模型（基于内容）

```
成绩分析 → 找出薄弱/优势课程
         → 匹配预定义推荐库 → 生成推荐列表
```

- **策略**：根据平均分区间 + 具体课程强弱项
- **来源**：预定义的课程/方法/资料推荐库
- **输出**：个性化推荐列表(含推荐理由)

## 4. 面试考点整理

### Java 相关

| 考点 | 说明 | 出现位置 |
|------|------|----------|
| Spring Boot 自动配置 | `@SpringBootApplication` 原理 | 入口类 |
| 依赖注入 | `@RequiredArgsConstructor` + `final` | 所有 Controller/Service |
| RestTemplate | 如何调用外部 API | `AiAnalysisServiceImpl` |
| MyBatis-Plus CRUD | `BaseMapper` 自动实现增删改查 | Mapper 接口 |
| CORS 跨域 | 前后端分离的跨域处理 | `CorsConfig.java` |
| DTO/VO 设计模式 | 数据传输对象 vs 视图对象 | `dto/` 包 |
| `@Value` 读取配置 | 从 yml 读取自定义属性 | `AiAnalysisServiceImpl` |

### AI / 机器学习相关

| 考点 | 说明 | 出现位置 |
|------|------|----------|
| 线性回归原理 | 最小二乘法、R²评估 | `performance_model.py` |
| KMeans 聚类 | 无监督学习、k值选择 | `cluster_model.py` |
| 情感词典方法 | 基于规则的NLP vs 深度学习方法 | `sentiment_model.py` |
| 协同过滤 | User-based vs Item-based | `recommend_model.py` |
| 特征工程 | 如何选择和处理特征 | 各模型的 `_prepare_features` |

### 数据库相关

| 考点 | 说明 |
|------|------|
| 索引设计 | 外键字段建索引 |
| 表关系设计 | student → performance (1:N) |
| 分页查询 | MyBatis-Plus Page 对象 |
| 时间字段 | `DATETIME` vs `TIMESTAMP` |

## 5. 扩展思路

### 短期扩展（易实现）

1. **图表可视化**：用 ECharts / Chart.js 渲染聚类饼图、成绩趋势折线图
2. **实时情感分析**：学员提交反馈时实时计算情感得分
3. **成绩排名**：班级内成绩排行、进步最快学员榜单
4. **PDF 报告导出**：使用 Apache POI 生成可下载的分析报告 PDF

### 中期扩展（需一定投入）

1. **深度学习模型**：使用 PyTorch/TensorFlow 实现 LSTM 成绩预测（比线性回归更准）
2. **自然语言处理**：改用 BERT 预训练模型做情感分析（远超词典方法）
3. **推荐系统升级**：基于协同过滤的矩阵分解（SVD）或神经网络推荐
4. **实时推送**：WebSocket 实时推送学情预警（成绩下滑、违纪增多）
5. **数据大屏**：教室电视上展示班级整体学情数据大屏

### 长期扩展（架构级）

1. **微服务拆分**：用户服务、教学服务、AI 服务独立部署
2. **消息队列**：RabbitMQ/Kafka 异步处理 AI 分析任务
3. **缓存层**：Redis 缓存热门查询和 AI 分析结果
4. **多租户**：支持多个学校/机构使用同一平台
5. **A/B测试**：不同推荐策略之间做效果对比

## 6. 项目亮点总结（用于答辩/面试）

1. **前后端分离**：纯前端静态页面 + RESTful API，易于扩展
2. **AI 增强**：Spring Boot + Python 双语言架构，Java 做主业务，Python 做 AI
3. **Docker 容器化**：一键部署，环境一致
4. **高可读性**：清晰的包结构，完善的注释，统一的错误处理
5. **边界处理**：数据不足情况下的降级策略（线性回归 → 平均估算）
6. **循序渐进**：保留原有 CRUD 功能，新增功能独立模块，不影响现有业务
